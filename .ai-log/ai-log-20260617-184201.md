# AI Work Log

## Objectives

Loại bỏ `createdAt` khỏi `VocabularyType` và các helper lưu trữ liên quan, nhưng giữ nguyên `id` để không phải mở rộng phạm vi sang logic sửa/xóa/kéo thả của Vocabulary.

## Plan

1. Update `VocabularyType` để bỏ field `createdAt`.
2. Update `src/lib/localStorage.ts` để:
   - đọc dữ liệu cũ an toàn hơn,
   - không còn tạo `createdAt` khi thêm mới,
   - đổi các helper thêm vocab sang kiểu không còn `createdAt`.
3. Update các chỗ đang dùng `Omit<VocabularyType, "id" | "createdAt">` sang kiểu mới.
4. Run lint để xác nhận không có lỗi TypeScript/ESLint phát sinh.

## Changed

- `src/types/vocabulary.ts`
  - Removed `createdAt` from `VocabularyType`.
- `src/lib/localStorage.ts`
  - Added safe normalization for vocabulary records loaded from `localStorage`.
  - Kept existing `id`-based behavior.
  - Removed `createdAt` generation from new vocabulary items.
  - Updated add helper signatures to `Omit<VocabularyType, "id">`.
- `src/components/vocabulary/vocabulary-table.tsx`
  - Updated import type for new vocabulary records to `Omit<VocabularyType, "id">`.
- `guide.md`
  - Updated Vocabulary model and CRUD/import-export notes to remove `createdAt`.

## Result

`createdAt` is no longer part of the vocabulary data model, while `id`-based edit/delete/drag behavior remains unchanged. Old stored records are still read safely, and new records are saved without `createdAt`.
